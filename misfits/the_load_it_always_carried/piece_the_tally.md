# Piece: The Tally

## Taxonomy

[The Load It Always Carried](play_the_load_it_always_carried.md). The linear accumulation to unity, the running sum across every cycle borne, no single-cycle check able to reveal how close it stands to one.

## Owner

- Project: khai-misfits

## Place

Read only at [the Fracture Surface](place_the_fracture_surface.md), after the fact, in the striations that count what no instrument counted while the part was in service.

## Load Bearing

Without it the increments would stay increments, each one spent and each one forgotten, and no cycle count would ever add to a verdict.
With it every increment the part has ever taken adds itself to the fraction before it, load of one size and load of another, all summing on the same linear scale toward the same number, one. The Tally is load bearing because it is the sum n over N carried to unity (Palmgren and Miner): the running total across loads of varying size, and it is load bearing precisely because it is the piece no single-cycle check reveals, the part passing every one of them while the sum it cannot see keeps climbing underneath.

## Apparent

Nothing, because there is nothing at the surface to sum: each cycle is checked alone, passes alone, and is filed alone, and no test asks what has already been spent.
Seen at the surface, the Tally does not exist, since the working life of the part shows a string of individually sound verdicts and nothing that adds them, and the truth that the string is not the story, that a hundred sound verdicts can still amount to a fraction one increment short of failure, stays hidden behind the fact that only the last cycle in the string ever looks different from the rest, and by then it is the last.

## Yearbook

Climbed by every [Increment](piece_the_increment.md) that [Accumulating](process_accumulating.md) adds, cycle upon [Cycle](piece_the_cycle.md), unseen through every pass of [the Passed Check](piece_the_passed_check.md).
It is the reckoning of the trap: the Tally is the sum that [the Fatigue Limit](piece_the_fatigue_limit.md) is set to keep from ever reaching one, and when it does reach one it is not read as a number but lived as [the Crack](piece_the_crack.md), the accumulation made visible only where it stops being invisible.
