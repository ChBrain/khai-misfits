# Process: Cycling

## Taxonomy

[The Load It Always Carried](play_the_load_it_always_carried.md). The single application of the working load, borne within limits and passed on every check, run again and again as the part's ordinary service, the process that begins the count and repeats it a thousand times before anything shows.

## Owner

- Project: khai-misfits

## Initiated by

A load below yield, met by a part that is sound.
At [The Rig](place_the_rig.md) or in [The Service](place_the_service.md), the Load applies its stress within every limit and the Part receives it, well inside the strength that would harm it, and nothing in the application itself is a fault.

## Direction

Toward one more cycle borne, safely, within limits.
Because the load never exceeds yield and the part is genuinely equal to it, cycling repeats the same safe application without variation, so it runs toward the next application and the one after, each identical to the last, the sameness itself the whole of the direction.

## Lever

The working load meeting metal that can bear it, every single time.
Because the stress sits below yield and the material is sound, each application is a fair contest the part wins outright, so cycling runs on a true and repeatable margin, the same safe load and the same safe answer, cycle after cycle, with nothing in the contest itself that changes.

## Echo

One more cycle spent, and nothing to show for it but the count.
The load has been borne and released and the part reads exactly as it did before, whole on the dial and sound on the check, so what looks like nothing happening is one increment added to a tally no test can see, the part one cycle nearer the sum and no instrument any wiser.
