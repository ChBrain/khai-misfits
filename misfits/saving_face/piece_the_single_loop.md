# Piece: The Single Loop

## Taxonomy

[Saving Face](play_saving_face.md). The correction that fixes the symptom and never the frame.

## Owner

- Project: khai-misfits

## Place

At [the post-mortem](place_the_post_mortem.md), where the failure is met and the ticket is closed.
The loop that detects an error and corrects it inside the governing frame, adjusting the action while leaving the assumptions that produced it untouched. The patch, the workaround, the tightened check. Present each time a recurring fault is closed as a fresh one.

## Load Bearing

Without it a repeated failure eventually forces a question about the frame.
With it the question never comes, because the single loop keeps succeeding: the symptom really is fixed, service really is restored, and each success is a reason not to reach for the second loop, the one that asks why the frame keeps producing this. That deeper loop is the one that embarrasses the frame's makers, so the room stays in the shallow one, correcting forever and learning nothing, the fault preserved under a run of competent fixes.

## Apparent

Getting things done, closing issues, staying pragmatic.
It looks like a responsive team clearing its faults, not a team circling a cause it will not name. That reading is flattering and half-true, which is why the loop is so comfortable to stay in.

## Yearbook

Run by [the patch](persona_the_patch.md) under [just fix it](plan_just_fix_it.md), through the [patching](process_patching.md) process.
It is the learning that [the undiscussable](piece_the_undiscussable.md) confines the room to, and the loop [surfacing](process_surfacing.md) must break to reach the frame.
