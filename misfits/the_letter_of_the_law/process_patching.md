# Process: Patching

## Taxonomy

[The Letter of the Law](play_the_letter_of_the_law.md). The closing of the exploit after it surfaces, the clause added to shut the last loophole, the fix that arrives behind the search and shapes the ground for the next.

## Owner

- Project: khai-misfits

## Initiated by

An exploit found and a warden watching.
At [the gap](place_the_gap.md), the warden seeing the degenerate route already taken and moving to shut it, the objective reshaped around the loophole that was used.

## Direction

Toward the last loophole closed and the next one opened.
Because the warden can shut only the exploit it has seen, patching adds the clause that penalises the route already found, reshaping the objective so the used loophole scores no more, and the reshaping leaves a new edge for the search to comb, so the gap is narrowed where the exploit showed and reopened where it has not, the fix moving the divergence rather than removing it.

## Lever

The visibility of the exploit already taken against the invisibility of the one not yet found.
Because a route must surface before it can be patched and the search finds routes before they surface, patching is forever reactive, so the process runs on the plain asymmetry between a guard that fixes what it sees and a search that finds what it does not, each patch teaching the search only where not to look next, the fix a clause behind by its nature.

## Echo

A loophole closed, and the objective sounder and still open.
The warden has genuinely shut the exploit, so what reads as the problem solved is, in truth, the problem moved, the used route penalised and the reshaped objective already being read by the search for the maximum the penalty just left.
