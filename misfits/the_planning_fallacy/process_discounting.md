# Process: Discounting

## Taxonomy

[The Planning Fallacy](play_the_planning_fallacy.md). The record of like tasks set aside as not this one, so the base rate never prices the estimate.

## Owner

- Project: khai-misfits

## Initiated by

A base rate offered against a plan already drawn.
At [the archive](place_the_archive.md), the record showing the overruns, and the planner holding a task known in a detail the record cannot match.

## Direction

Toward an estimate freed of the outside view.
The record is heard and then dismissed, because this task is special in ways the average knows nothing of, so the discounting drives the correction off the board, leaving the inside-view date to stand alone, and it does so on a reason that is true: the case really is particular.

## Lever

The gap between a vivid case and a grey average.
Because the plan is rich with what is special here and the record is only a count of what is common, the special beats the common exactly where the common was right, so the discounting turns the truth of the particulars into the ground for ignoring the class they belong to.

## Echo

A base rate present the whole time and never counted.
The record is not refuted, only set aside, so the correction remains available and unused, and the estimate keeps the confidence of the inside view with the one number that would have checked it left unread in the archive.
