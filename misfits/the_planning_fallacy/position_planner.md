# Position: Planner

## Taxonomy

[The Planning Fallacy](play_the_planning_fallacy.md). The seat that lays the task out and estimates it from its own plan.

## Owner

- Project: khai-misfits

## Has

The task in front of it, and a plan of the steps to do it.
The detail of this case, vivid and truly known, to reason from.
No weight on the record of tasks like it, only the run it can draw.

## Orders

Lay the work out and estimate it.
Sum the parts at about the time each should take, and reach the date.
Read a plan thought through as an estimate to trust.

## Loses

The truth of the number, exactly by reasoning from the plan.
Because a plan traces the run where the work goes well, the estimate it yields is the best case, and the seat cannot see this, because the best case is the only case the plan draws.
The more carefully it works from this task's own steps, the further the base rate of like tasks falls from view.

## Drives

Toward a date built from the inside view,
felt as right because it was thought through,
so the diligence that knows the task best prices it worst.
