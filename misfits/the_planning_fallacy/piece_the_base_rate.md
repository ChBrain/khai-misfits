# Piece: The Base Rate

## Taxonomy

[The Planning Fallacy](play_the_planning_fallacy.md). The distribution of how like tasks really went, the outside view made into an object, present and set aside.

## Owner

- Project: khai-misfits

## Place

At [the archive](place_the_archive.md), where the record of tasks already done is kept, estimate against actual.
Carried, when it is carried at all, to [the reforecast](place_the_reforecast.md), where the estimate is set on it in place of the plan.

## Load Bearing

Without it the inside view has nothing to be corrected by, and the fallacy is invisible to itself.
With it the estimate could be priced right: the base rate is the class the task belongs to, its outcomes counted, its overruns one-directional and plain. The Base Rate is load bearing because it is the correction that is always available and almost never used (Kahneman and Tversky; Flyvbjerg): the trap is not that the record is missing but that it is discounted as not this case, so the piece carries both the answer and the reason the answer is refused, its own grey generality.

## Apparent

An irrelevance, a set of other tasks that are not this one.
Seen from the board, the base rate looks like it cannot know what is special here, and the truth that what is special is exactly the detail that misleads, and that the average is right where the plan is wrong, stays hidden behind how plainly this task differs from the ones in the record.

## Yearbook

Held by [the record](persona_the_record.md) and shown through [discounting](process_discounting.md), the outside view offered and waved off.
It is the class against the case: the Base Rate is what [the archive](place_the_archive.md) keeps that [the drawing board](place_the_drawing_board.md) leaves out, the number [the forecaster](persona_the_forecaster.md) would set the estimate on, and the count in which [the overrun](piece_the_overrun.md) was written before this task began.
