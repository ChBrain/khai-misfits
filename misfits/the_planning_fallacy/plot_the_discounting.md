# Plot: The Discounting

## Taxonomy

[The Planning Fallacy](play_the_planning_fallacy.md)

## Owner

- Project: khai-misfits

## Cue

[The Archive](place_the_archive.md), the record of how long tasks like this really took, opened beside the plan.
Every line running the same way, the actual past the estimate, the base rate plain to anyone who reads it.

## Action

[Discounting](process_discounting.md): [The Record](persona_the_record.md), holding [the record's seat](position_record.md), offers the base rate, while [The Planner](persona_the_planner.md), holding [the planner's seat](position_planner.md), sets it aside as not this task.

## Stage

[The Record](persona_the_record.md), having tabled [Show the Record](plan_show_the_record.md), lays the overruns of like tasks beside the new date and asks that they price it.
[The Planner](persona_the_planner.md) holds to [Plan the Work](plan_plan_the_work.md), answering that this task is known in a detail the average cannot see, and [The Sponsor](persona_the_sponsor.md), on [Set the Date](plan_set_the_date.md), needs the low number to stand.

## Tension

The correction is on the table, and the reason to refuse it is true.
There is no denial to catch: the record really does not know what is special here, and this task really is particular. But the particulars are exactly the inside view that misleads, and the average is right where the plan is wrong, so the discounting turns a true fact about the case into the ground for ignoring the class it belongs to, and the one number that would have checked the estimate is left unread in the archive.
