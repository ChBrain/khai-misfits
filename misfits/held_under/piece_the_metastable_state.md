# Piece: The Metastable State

## Taxonomy

[Held Under](play_held_under.md). The bad equilibrium: the failing state the system sticks in once tipped, stable enough to persist on its own and requiring force, not time, to leave.

## Owner

- Project: khai-misfits

## Place

Held at [The Queue](place_the_queue.md), which is where the sustaining feedback actually lives: the state is not a property of any one server or request, it is the condition of the queue staying full because everything leaving it failed returns retried. It is what [The Steady State](place_the_steady_state.md) is not: the second, latent rest point the system was one push away from the whole time.

## Load Bearing

It is load bearing as the structural claim the entire misfit turns on: the system has two stable states, the healthy one and a latent bad one beneath it, and a trigger crossing the threshold between them tips the system into the bad state, where a sustaining effect, distinct from the trigger, holds it there after the trigger is gone (Bronson, Charapko, Aghayev, and Zhu). This is what separates the misfit from ordinary congestion: a congested system is simply busy and recovers the moment the extra load eases, but a metastable one does not recover even once the original load is long past, because the state itself, not the load that opened it, is now what needs to be broken. Precision matters here: the state persists specifically because it is sustained by correct recovery behaviour, not because anything remains broken.

## Apparent

Looks like an ongoing overload event, a system still fighting the same fire that started it. The apparent continuity is misleading: the fire that started it is out, and a different one, made of the system's own retries, is what is actually burning.
