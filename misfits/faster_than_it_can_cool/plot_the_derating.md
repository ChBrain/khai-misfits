# Plot: The Derating

## Taxonomy

[Faster Than It Can Cool](play_faster_than_it_can_cool.md).

## Owner

- Project: khai-misfits

## Cue

At [The Diagram](place_the_diagram.md), and against [The Rating](place_the_rating.md) it will be written onto, the threshold is finally read plainly: the two curves, the distance between them, and the single point where the line only touches the curve.

## Action

[The Engineer](persona_the_engineer.md), on [Hold the Margin](plan_hold_the_margin.md), reads the Diagram and derates the operating point far below [the Critical Point](piece_the_critical_point.md): adds cooling, fits a vent, sets [the Rating](place_the_rating.md) with a wide printed distance to the threshold. Every setting works, and every setting costs, paid in the capacity, energy, and mass given up to buy the distance.

## Stage

This is done correctly. The Engineer computes the tangency, not a guess but a hard value read straight off the curves, and sets the operating point well clear of it, exactly as a careful reading of the curve requires. The margin held is real, the cooling added really cools, the vent really vents. Nothing here is negligence, and nothing here is free: capability is spent, on purpose, for distance.

## Tension

The margin is held and the cost is real and permanent, and [Drifting](process_drifting.md) is still narrowing it. Holding the margin buys time against the ageing and never stops it, because no derating touches the asymmetry underneath: an exponential source will always eventually out-climb a linear sink, whatever distance is bought between them today. The Engineer does everything right, and everything right holds the point off for a while and never removes it, so the tax is owed again every year the system runs.
