# Piece: The Network

## Taxonomy

[The Extra Road](play_the_extra_road.md). The graph of routes from one end to the other, whose times depend on how many drive them.

## Owner

- Project: khai-misfits

## Place

At [the map](place_the_map.md), the whole set of links seen at once: the roads, the junctions, the loads.
Not one route but every route together, and the property that makes the paradox possible: some links cost the same however busy, and some cost more the more they carry, so the network's total time is not fixed but is made, link by link, from the choices poured into it.

## Load Bearing

Without it there is no whole for a local choice to harm, and adding a road could only ever help.
With it there is a system whose time is the sum of load-dependent parts, so a change that is good at one link can be bad across all of them. The Network is load bearing because Braess lives in the difference between a link and the graph: a driver feels a link, the network feels the load, and only because the two are not the same can more capacity buy less flow.

## Apparent

Infrastructure. Roads, capacity, options, all plainly good to have.
It looks like the more of it the better: more links mean more ways through, more slack, more resilience, so a network reads as a thing you improve by adding to. That reading is the trap, because it counts links and not loads, and the network's time was never a count of roads.

## Yearbook

Shaped by [the planner](persona_the_planner.md) who adds to it and by [routing](process_routing.md) that fills it.
It is the field every other piece plays on: [the jam](piece_the_jam.md) is a state of it, [the extra road](piece_the_extra_road.md) is an addition to it, and [the equilibrium](piece_the_equilibrium.md) is the load it settles into once every driver has chosen.
