# Plot: The Holdback

## Taxonomy

[No Idle Hands](play_no_idle_hands.md)

## Owner

- Project: khai-misfits

## Cue

[The Reserve](place_the_reserve.md), someone unwilling to spend the last of the slack.
The Reserver setting a target busy fraction below saturation and defending the idle time against the pressure to load it.

## Action

[Reserving](process_reserving.md): [The Reserver](persona_the_reserver.md), holding [the reserver's seat](position_reserver.md), keeps planned slack and refuses the last increment that reads most efficient and buys the choke.

## Stage

The Reserver, on [Hold the Slack](plan_hold_the_slack.md), keeps [the slack](piece_the_slack.md) as the buffer for [the variability](piece_the_variability.md) and holds [the utilisation](piece_the_utilisation.md) below [the saturation](piece_the_saturation.md), so [the wait](piece_the_wait.md) stays bounded and [the jam](piece_the_jam.md) never forms.
The idle time is left free on purpose, against a metric that reads it as waste and rewards the fraction the Reserver refuses to raise.

## Tension

Holding slack is the one move that keeps the flow fast, and it is exactly the idle capacity the manager is right to hate.
There is nothing free in it: the Reserver must tolerate visible idleness, refuse the increment that reads best, and answer for an idle hour whose worth is a jam that never happens. The choke it prevents leaves no trace, so the plot closes with the slack held and the flow kept fast ... or the idle capacity loaded as waste, the buffer spent, and the wait left to diverge for a metric that reads well up to the wall.
