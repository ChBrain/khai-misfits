# Place: The Reserve

## Taxonomy

[No Idle Hands](play_no_idle_hands.md). Where slack is held on purpose, the one arrangement in which the resource is left idle deliberately so the buffer stays to absorb the variability and the wait stays bounded.

## Owner

- Project: khai-misfits

## Shown

The place capacity is kept free on purpose, a target utilisation set below one, protected slack, a buffer defended against the pressure to load it.
The Reserver holding the busy fraction below saturation, leaving idle time deliberately, and the variability finding somewhere to sit so the line behind the server stays short.

## Holds

The planned slack, and the flow it keeps fast.
Here capacity is left free before it is loaded, so the Reserve holds the point where the buffer is kept: idle time held on purpose so the variability has somewhere to sit, the busy fraction kept below the wall, so it is where the slack is defended rather than spent, the one arrangement in which the wait stays bounded because the idle time was never squeezed away.

## Offers

A way to keep the wait bounded, and the flow fast rather than lean.
Because the Reserve keeps the slack the variability needs, it offers the one thing the Floor could not, the buffer held so the line never diverges, so it hands out a flow kept fast, and hands it out at the price of tolerating visible idleness and refusing the last increment that would have read best on the metric.

## Withheld

Any leanness on the metric, or any proof the slack was needed.
The Reserve can keep the flow fast but cannot make the held capacity read as anything but idle, because the jam it prevents leaves no trace and the utilisation it holds down is plain, so it withholds the look of a resource well run, exacting the appearance of tolerating waste, and, all bounded wait and no visible payoff, it withholds the thanks for a choke that now never comes, the idle hour seeming, after, like capacity that was simply not used.
