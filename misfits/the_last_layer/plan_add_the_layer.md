# Plan: Add the Layer

## Taxonomy

A **standing plan**: the intent of [the Office](position_the_office.md), inside [The Last Layer](play_the_last_layer.md).

## Owner

- Owner: [The Office](position_the_office.md)

## Direction

Meet the problem by raising the structure that solves it. Answer each difficulty with the cheapest effective complexity available, and keep that structure standing, because a solved problem stays solved only while its layer remains. Take the easy, high-return solutions first and the method proves itself, so the next problem is met the same way, no thought of a curve required while the return is still large.

## Orders

([The Administrator](persona_the_administrator.md) raises a new layer of complexity against the problem at hand and is rewarded, the early return large and the cost small.)

## Implementation

([The Frontier](place_the_frontier.md); the [complexifying](process_complexifying.md) process; the Administrator adds [the layer](piece_the_layer.md) that solves the problem and banks [the return](piece_the_return.md), the solution kept and fed as standing [upkeep](piece_the_upkeep.md) thereafter.)

## Targets

- [ ] the problem at hand is met by raising the structure that solves it
- [ ] the layer that solves the problem is kept standing, not dismantled
- [ ] the cheapest effective solution is taken first, so the early return is large
- [ ] the method, having worked, becomes the standing answer to the next problem
