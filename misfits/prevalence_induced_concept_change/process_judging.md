# Process: Judging

## Taxonomy

[Prevalence-Induced Concept Change](play_prevalence_induced_concept_change.md). The comparative reading of each case against recent company, sound perception and the engine of the drift.

## Owner

- Project: khai-misfits

## Initiated by

A task to find the thing and a run of cases to weigh.
At [the bench](place_the_bench.md), the assessor reading each new case in the light of the recent ones, calling the thing and passing the fine.

## Direction

Toward a verdict on each case, set by the company it keeps.
The assessor weighs the new case against the recent run rather than an absolute rule, so judging draws every ruling toward the current context, and where the context sits is where the concept's edge is drawn.

## Lever

The soundness and ease of judging by context.
Because reading each case in the light of the recent ones is how perception works and mostly serves us well, the comparative call feels natural and strict at once, so judging runs on good perception, the drift built entirely from judgements no one can fault.

## Echo

A fair verdict on each case, tied to a moving reference.
The assessor has judged honestly and feels its standard fixed, and every ruling is quietly pinned to the recent company, so what reads as consistent judgement is, unseen, a standard that will travel with whatever the run becomes.
