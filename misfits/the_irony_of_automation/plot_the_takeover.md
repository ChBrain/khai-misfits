# Plot: The Takeover

## Taxonomy

[The Irony of Automation](play_the_irony_of_automation.md)

## Owner

- Project: khai-misfits

## Cue

[The Floor](place_the_floor.md), the routine work ready to be mechanised and a machine that will run it better.
The dull, repetitive part handed over, the operator freed to oversee rather than do.

## Action

[Automating](process_automating.md): [The Automation](persona_the_automation.md), holding [the automation's seat](position_automation.md), takes the routine and runs it reliably, while [The Designer](persona_the_designer.md), holding [the designer's seat](position_designer.md), automates everything that can be and leaves the human the rest.

## Stage

[The Automation](persona_the_automation.md), having tabled [Take the Routine](plan_take_the_routine.md), runs the frequent work faster and cleaner, and asks only to be overseen.
[The Designer](persona_the_designer.md), on [Leave the Rest](plan_leave_the_rest.md), mechanises the tractable cases and hands the operator the residue, reading the automated system as the job done well.

## Tension

A real improvement, and it quietly ends the operator's practice.
There is nothing to fault: the machine genuinely does the routine better, freeing the human from drudgery is a gain, and automating the tractable is sound design. But the routine taken was also the operator's daily rehearsal, so the plot closes with the system improved, the operator turned from doer to monitor, and the practice that kept their skill gone with the work, none of it yet visible as a loss.
