# Play: The Irony of Automation

## Estate

[khai-misfits](https://github.com/ChBrain/khai-misfits): the house that holds this production and answers for the whole run.

## Name

The Irony of Automation.
The machine takes over the routine work, and it does it well: fewer slips, less toil, the operator freed from the dull repetitive part to simply watch over it. And that is exactly the trouble. The routine was also the practice that kept the operator's hand in, so as the machine does more the operator does less, and the skill quietly rusts. And watching a machine that almost never fails is a thing humans are poor at, so the operator drifts, half-attentive, out of the loop. Then one day the rare case the machine was never built to handle arrives, and it hands control back ... to an operator who is rusty, disengaged, and facing the single hardest problem in the whole system, cold. The better the automation, the more it takes, and the less able the human it leaves to catch what it drops.

## Arc

The bend is this: automating the routine is a real improvement, and the same improvement erodes the very competence the system will need when the automation reaches its limit.
It runs on what the routine was for.
The frequent, easy work is the obvious thing to automate: the machine does it faster and with fewer errors, and the operator is spared the drudgery. But that same routine was the operator's rehearsal, the daily handling that kept a skill sharp, so removing it removes the practice, and a skill without practice fades. The operator who once flew the whole flight by hand now watches an autopilot, and the hand forgets.
And watching is its own trap. A human set to monitor a system that rarely does anything wrong cannot stay sharp on it: attention wanders, trust grows, and the operator settles out of the loop, no longer tracking the state closely enough to step in fast. So the automation produces, at once, an operator whose skills have rusted and whose attention has drifted, and it does this precisely by working well, because the better and more reliable it is, the less the human practices and the less there is to watch.
None of it is error or sloth. Automating the routine is sound, the machine really is better at it, freeing the human from drudgery is a gain, and no one can stay vigilant on a system that almost never fails or keep a skill sharp with nothing to practise on. The operator is not lazy and the designer not careless: the designer automated everything that could be automated, and what is left for the human is exactly the residue too hard or too rare to automate, the worst cases, handed over at the worst moment to the least ready hand.
Keep the operator in the loop, drill the skills the automation no longer exercises, hand control back before the crisis and not in it, design the machine to keep its human engaged, and the hand can stay in: but the practice costs the very efficiency the automation was bought for, engaging a human who is not strictly needed looks like waste, the skill can never be fully kept without doing the work the machine now does, and no drill quite matches the real rare case, so the competence that would catch the failure is bought back only by giving up part of what the automation was for.

## Stakes

Whether the human left to catch what the machine drops is ever ready to catch it.
Every plot but the last moves the same way: toward the automation, which each time genuinely improves the routine, and away from the practice, which costs the efficiency the automation was bought for and looks like waste on every day the machine does not fail. The production cannot make the operator sharp on a system it no longer works, because the routine that would have kept the skill and the failures that would have kept the attention are exactly what the automation removed, so the competence fades in proportion to how well the machine runs. It can only show that no one was lazy or careless, that automating the routine is sound and freeing the human is a gain and no one stays vigilant on a machine that never fails, and that the one thing that keeps the hand in, drilling and engaging a human the automation has made all but unnecessary, must be paid for in the very efficiency the automation promised, so the machine is better and the human is worse and the gap between them is widest exactly when the rare failure comes.
