# Process: Automating

## Taxonomy

[The Irony of Automation](play_the_irony_of_automation.md). The routine handed to the machine, a real improvement that removes the operator's practice.

## Owner

- Project: khai-misfits

## Initiated by

A routine task that can be mechanised and a designer to mechanise it.
At [the floor](place_the_floor.md), the designer automating every tractable case and the machine taking the frequent work off the operator's hands.

## Direction

Toward a system that runs the routine better and a human left to oversee it.
Each task automated is done faster and with fewer slips, so automating drives a genuine gain, and because the routine it takes was also the operator's rehearsal, it drives at the same time the removal of the practice that kept the skill, improvement and erosion in the one act.

## Lever

The machine's real superiority at the routine.
Because the automation genuinely does the frequent work better, every case handed to it is a sound decision, so automating runs on true improvement, and the erosion it causes rides along invisibly, never a reason to stop and always a consequence of going on.

## Echo

A routine run well and a hand with nothing left to do.
The system is better and the operator freed, and the practice that kept that operator sharp is gone with the work, so what reads on the floor as pure progress is, unseen, the start of the rust and the drift the handover will one day meet.
