# Process: Adding

## Taxonomy

[Belt and Braces](play_belt_and_braces.md). The Designer placing a backup behind a single point, the good-faith addition that closes a real gap and does not see the coupling it brings.

## Owner

- Project: khai-misfits

## Initiated by

A real single point of failure and a spare to remove it.
At [the design](place_the_design.md), the designer seeing the gap and placing a backup behind it, computing the parallel reliability.

## Direction

Toward the gap closed and the figure climbing.
Because the single point is real and the arithmetic of parallel redundancy rewards a spare, adding places the backup and reads the higher number as a safer system, so the defence in depth goes in plainly and the coupling it brings goes unpriced, the designer seeing the failure caught and not the interactions opened or the cause shared.

## Lever

The soundness of the arithmetic for independent parts.
Because a parallel backup does raise the computed reliability when the parts fail independently, adding runs on real reliability theory and not on carelessness, so the coupled addition is assembled from no error, only from a designer applying a correct arithmetic to a part the coupled system will not let be independent.

## Echo

A single point removed, and a coupling brought in.
The designer has closed a real gap as soundly as the figure says, so what reads as a system made safer is, in truth, one more part inside the system, one more path between the parts and one more shared cause, the backup placed and the turn it begins unseen.
