# Process: Coupling

## Taxonomy

[Belt and Braces](play_belt_and_braces.md). The added backup opening interactions the designers did not draw and resting on a cause it shares, the growth of complexity and common mode that turns redundancy into a single point.

## Owner

- Project: khai-misfits

## Initiated by

A backup placed inside the system, wired and cooled and crewed with the rest.
At [the system](place_the_system.md), the spare sitting among the parts it must sit among to back them.

## Direction

Toward the interactions multiplied and the cause shared.
Because a backup sits inside the system and not apart from it, coupling opens one more path between the parts with each addition and rests each spare on the power, crew, premise or environment it shares with what it backs, so the more redundancy there is the more complexity there is and the more cause is held in common, the many chances the arithmetic counted independent wired quietly into one.

## Lever

The necessity of sitting inside the system.
Because a backup apart from the system could not back it, coupling runs not on any oversight but on the plain requirement that a spare be wired and cooled and crewed where the primary is, so the interactions and the shared cause are assembled from no carelessness, only from redundancy placed where redundancy must be placed to be redundancy.

## Echo

More cover, and more coupling under it.
The system has gained a real spare exactly where it needed one, so what reads as defence in depth is, in truth, a set of parts sharing more paths and more causes with every addition, the redundancy grown and the independence spent, the common mode latent and waiting for the one cause to fire.
