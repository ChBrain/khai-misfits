# Piece: The Reliability

## Taxonomy

[Belt and Braces](play_belt_and_braces.md). The apparent reliability figure that reads higher with each backup added, the number computed from parallel independence that climbs while the true reliability, uncounted, has begun to turn down.

## Place

## Load Bearing

Without it the additions have no reward and no one mistakes more spares for more safety, and the turn would be plain.
With it the trap has its deceptive metric: the figure that rises with every parallel unit as if each were independent, so each backup reads on the page as more safety even where the coupling and common cause and shirking are lowering the true reliability the figure cannot see (the arithmetic of parallel redundancy computed as though the parts fail independently, reporting more safety exactly where the coupled system holds less). The Reliability is load bearing because it is what keeps the additions coming: a number that only climbs, taken for the reliability it names, so the piece carries the metric that reads highest just as the system turns, the count of spares mistaken for the count of chances.

## Apparent

The reliability of the system, more nines with every backup, the safety plainly measured and plainly rising.
Seen at the surface, the Reliability looks like the truth about the system, a figure any sound method would compute and that rises for good reason, and the truth that it counts spares and not independence, that it assumes an independence the coupled system denies, that it climbs while the reliability it names turns down, stays hidden behind how much a rising number looks like a safer system.
