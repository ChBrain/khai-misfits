# Piece: The Overrun

## Taxonomy

[Belt and Braces](play_belt_and_braces.md). The phenomenon whole, the point past which each further backup lowers the true reliability it was added to raise, the redundancy that turns from cover to cost while the figure still climbs.

## Owner

- Project: khai-misfits

## Place

Reached across [the system](place_the_system.md), where the additions have carried the reliability past its peak.
Forestalled, if ever, at [the review](place_the_review.md), where the turn is counted before the common cause finds it.

## Load Bearing

Without it more redundancy is simply more safety and there is no paradox, only the classic arithmetic holding.
With it the trap is complete and authorless: a system past the point where the coupling, common cause and shirking each backup brings outweigh the cover it adds, so each further spare subtracts from the true reliability while the computed figure still rises (Sagan, on the redundancy problem, more nuclear security forces producing less nuclear security past the point where added guards add more shared cause and shirking than independent catch). The Overrun is load bearing because it is the phenomenon whole: not one bad backup but the turn of the curve, the redundancy that was raising the reliability now lowering it, so the piece carries the peak already passed, the true safety declining as the defence in depth deepens and the figure climbs.

## Apparent

An ever safer system, redundancy deepening and the reliability rising with it, defence in depth working as it should.
Seen at the surface, the Overrun looks like exactly its opposite, a system made steadily safer by steadily more backups, and the truth that the curve has turned, that past the peak each addition subtracts, that the deepening defence is thinning the reliability while the figure reports the reverse, stays hidden behind how surely more redundancy reads as more safety, right up until the common cause fires.
