# Piece: The Complexity

## Taxonomy

[Belt and Braces](play_belt_and_braces.md). The new interactions each addition opens, the paths between the parts the designers did not draw, invisible because each backup was added to a diagram that shows the cover and not the coupling.

## Owner

- Project: khai-misfits

## Place

Opened at [the system](place_the_system.md), where the parts sit together and interact.
Priced at [the review](place_the_review.md), where the complexity each addition brings is counted against the cover it adds.

## Load Bearing

Without it a backup would be pure cover and the redundancy pure gain, and nothing unforeseen would grow.
With it the trap has its room: the interactions each added part opens, which multiply faster than the parts and reach beyond what any diagram drew, so a system tightly coupled fails in ways no one designed (Perrow, on interactive complexity and tight coupling, the added component multiplying the unexpected interactions that produce normal accidents). The Complexity is load bearing because it is where the unforeseen failure lives: not the failure the backup was added to catch but the new one the backup created by being one more part, so the piece carries the surface for failure that grows with every spare, seen by no one because each addition looks local and only the whole is coupled.

## Apparent

Nothing, on the surface: one more sound part in a well drawn system, each addition local and understood.
Seen at the surface, the Complexity is invisible, because each backup was added on its own to close its own gap and the interactions it opens are not on the diagram, so the system looks like the sum of understood parts and the coupling looks not to exist, and the truth that the parts interact along paths no one drew, that each addition multiplies them, that the whole is more coupled than any part, stays hidden behind how sound each local addition is.

## Yearbook

Opened by [the backup](piece_the_backup.md) through [coupling](process_coupling.md), priced only by [the auditor](persona_the_auditor.md) through [auditing](process_auditing.md), the paths the diagram did not draw.
It is the room of the trap: the Complexity grows with each addition to [the backup](piece_the_backup.md) set, hides beneath the climbing [reliability](piece_the_reliability.md), and its unforeseen interactions join [the common mode](piece_the_common_mode.md) in the failure [the overrun](piece_the_overrun.md) names.
