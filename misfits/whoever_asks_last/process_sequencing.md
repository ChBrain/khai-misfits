# Process: Sequencing

## Taxonomy

[Whoever Asks Last](play_whoever_asks_last.md). The setting of the order in which the pairings are put, which turns the headless cycle into a single winner and makes that winner a function of the order and not the room.

## Owner

- Project: khai-misfits

## Initiated by

[The Convenor](persona_the_convenor.md), at [the Order Paper](place_the_order_paper.md), fixing which options meet first and which the survivor faces next.
Begins when a winner must be found and the options can be taken only a pair at a time.

## Direction

Turns a cycle with no winner into a winner with no mandate.
Because the options are voted pairwise in some order and each contest eliminates one, sequencing converts the loop into a survivor, but the survivor depends wholly on the sequence, so the process manufactures a definite outcome from an indefinite group preference, the same honest votes returning a different winner for every running order and the coherence supplied by the order alone.

## Lever

The path-dependence of pairwise elimination.
Because in a cycle each option can be beaten by some other, the one that survives is the one the sequence never exposes to its conqueror until it has done its work, so sequencing works through the order of elimination: the lever is the running order itself, which selects the winner by choosing what meets what and when, an agenda-setter steering sincere voters to any option by the schedule and no vote changed.

## Echo

A declared winner, chosen by the order and binding on the room.
The echo of sequencing is a single option left standing, handed to the assembly as its decision, so the room carries an outcome fixed by whoever set the pairings and answerable to no collective preference, a result reached fairly in every contest and authored entirely by the sequence, decisive and legitimate and nobody's will but the schedule's.
