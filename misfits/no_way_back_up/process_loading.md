# Process: Loading

## Taxonomy

[No Way Back Up](play_no_way_back_up.md). The irreversible rise of the minimum mutational load, the click that follows each loss of a least-loaded class, the floor stepping up with no route by which it could step down.

## Owner

- Project: khai-misfits

## Initiated by

[The Toss](persona_the_toss.md), at [the Colony](place_the_colony.md), the floor rising to the next class each time the cleanest is dropped.
Begins when the least-loaded class is gone and the fewest mutations any lineage now carries is one more than before.

## Direction

Turns a lost class into a raised floor.
Because the population's minimum load is defined by its cleanest surviving class, and that class cannot be rebuilt by copying or restored by a reversal that will not come, the loss of the least-loaded steps the floor up by one, so loading converts a chance removal into a permanent rise, the fewest-mutations number climbing a notch that can only hold or climb again (Muller).

## Lever

A floor with only one direction left.
Because copying cannot lower the load and back-mutation is too rare to reach it, the minimum has no way down once its lowest class is gone, so loading works through the closing of every descent: the lever is the irreversibility itself, which makes each loss a ratcheting rather than a dip, the floor holding at its new height because nothing available can bring it back below.

## Echo

A minimum load one notch higher, and the same exposure waiting.
The echo of loading is a floor risen and fixed, with the class that is now cleanest again the rarest and again one unlucky draw from gone, so the climb is set to repeat, each click leaving a higher floor and a new smallest class exposed to the next, the load mounting one irreversible step at a time and, in a small population, quickening toward a weight the lineage cannot carry (Lynch, Bürger, Butcher and Gabriel).
