# Process: Regulating

## Taxonomy

[The Simpler Lever](play_the_simpler_lever.md). The Regulator meeting each disturbance with the response that cancels it, the matching of a repertoire against a field that holds the outcome for as far as the variety reaches.

## Owner

- Project: khai-misfits

## Initiated by

A disturbance arriving and a repertoire to answer it.
At [the console](place_the_console.md), the regulator reaching for the state that cancels the disturbance in front of it and keeping the essential variable in bounds.

## Direction

Toward the disturbance cancelled and the outcome held.
Because a distinct response absorbs a distinct disturbance, regulating pairs each thing the world does with the state that answers it, so the outcome stays within bounds while the repertoire covers the field, the control real and running exactly as designed for as far as the count of responses reaches.

## Lever

The distinct state ready for each distinct disturbance.
Because only a difference in response can absorb a difference in disturbance, regulating runs on the repertoire being at least as wide as the field, so the control is not attention or skill but a count: a state held for each thing that can arrive, and an outcome held only so far as those states are there to reach for.

## Echo

A disturbance cancelled, and an outcome still held.
The regulator has met what arrived with a matching state, so what reads as a controller simply working is, in truth, a repertoire that still happens to cover the field, holding the outcome until a disturbance comes for which the trimmed menu keeps no answer.
