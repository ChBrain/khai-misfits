# Plot: The Failed Check

## Taxonomy

[The File Drawer](play_the_file_drawer.md)

## Owner

- Project: khai-misfits

## Cue

[The Bench](place_the_bench.md), a doubter running the published study again, carefully and at power, and getting a null.
A replication that is the correction the record needs, arriving at the door of [the journal](place_the_journal.md) as a non-finding.

## Action

[Hardening](process_hardening.md): the failed replication is a null the same filter drawers, so the false positive stands and the record sets.

## Stage

[The Replicator](persona_the_replicator.md), on [Try to Correct](plan_try_to_correct.md), brings the failed check to the record, where it is [a null](piece_the_null.md) screened by the screen that kept the original, so [the significant](piece_the_significant.md) false positive stands in [the record](piece_the_record.md), cited as settled while its refutation gathers dust.
The check is sound, and the check found nothing, and the filter does not ask which.

## Tension

Running the study again is the science's own self-correction, and the self-correction is a null the record drawers like any other.
There is no flaw in the check: it was run at power and the effect was not there, the plain finding of an absence. But a failed replication is a non-finding, and nulls do not print, so the plot closes with the correction refused for the very reason it matters and the false positive left standing, a self-correcting science unable to correct through a filter that keeps only the positive, the error load-bearing precisely because its refutation could not enter.
