# Process: Hardening

## Taxonomy

[The File Drawer](play_the_file_drawer.md). The failed replication drawered by the same filter, so the false positive stands and the record sets into consensus.

## Owner

- Project: khai-misfits

## Initiated by

The Replicator, with a doubt and a null the check produced.
At the door of [the journal](place_the_journal.md), the failed replication arrives as a non-finding, and the same screen that kept the original meets the correction that would undo it.

## Direction

Toward a false positive locked into the record.
Because a failed replication is a null and nulls do not print, hardening points the correction into the drawer: the check that would overturn the finding is refused for the very reason it matters, that it found nothing, so the original stands unchallenged in the record while the challenge sits unseen, and the finding gathers citations as settled while its refutation gathers dust.

## Lever

The same filter, turned on the correction.
Because the screen keeps only the significant, it drawers the check exactly as it drawered every other null, so hardening runs on no new mechanism, only the original filter meeting the self-correction and treating it as the non-finding it is, no one needing to suppress the replication, only to prefer the finding to its failure, the science's own correction filtered out with everything else that found nothing.

## Echo

A record that cannot correct itself, hardening into consensus.
What leaves this process is a false positive standing uncontested and cited as established, its correction filed in a drawer, so the self-correcting science cannot self-correct through a screen that keeps only the positive, and the record sets into a confidence that no null was allowed to temper, the error load-bearing precisely because its refutation could not enter.
