# Piece: The Backward Rule

## Taxonomy

[Good Money After Bad](play_good_money_after_bad.md). Deciding on what has been spent, not on what is left to gain.

## Owner

- Project: khai-misfits

## Place

At every gate at once, the rule the room actually runs on.
Not written anywhere, and applied everywhere: the question the table asks is how much have we already put into this, when the question that decides well is how much more will it cost against how much it will return.

## Load Bearing

Without it a gate compares the cost still to come with the value still to get, and the failing project dies on schedule.
With it the gate compares the pile already spent with the shame of calling it waste, and the failing project lives, because the Backward Rule funds a thing in proportion to how much it has already failed. It is the single inversion the whole play turns on: a correct decision faces forward, and this one faces back, so the more evidence of failure accumulates, the more committed the room becomes.

## Apparent

Prudence, or loyalty, or resolve.
It looks like not being wasteful with what has been invested, not giving up too easily, honouring a commitment. Every name it wears is a virtue. Underneath every name it is the same reversal.

## Yearbook

Run by [the approver](persona_the_approver.md), felt by all, named by none.
It never appears on a slide, because no one would defend it stated plainly, and it governs every slide, because stated plainly is the one way it is never asked. [The good money](piece_the_good_money.md) follows the bad wherever the Backward Rule points, which is always deeper in.
