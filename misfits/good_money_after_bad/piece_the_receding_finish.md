# Piece: The Receding Finish

## Taxonomy

[Good Money After Bad](play_good_money_after_bad.md). The completion that moves out by exactly the time that passes.

## Owner

- Project: khai-misfits

## Place

On [the timeline](place_the_timeline.md), always the same distance ahead.
The milestone re-drawn each review to sit just beyond reach: one more phase, one more quarter, ninety percent done for the third year running.

## Load Bearing

Without it the failure would arrive on a date and be undeniable.
With it the failure never arrives, because the finish retreats as fast as the work advances, and a project that is always almost done is never quite done enough to kill. The Receding Finish is what keeps the payoff perpetually just close enough to justify one more phase and never close enough to collect, so the reason to continue renews itself every time it is about to expire.

## Apparent

Progress.
Each re-forecast is reasonable on its own: a fresh obstacle, a revised estimate, a plan to close the gap. It always looks like the last stretch. It is always the last stretch, and there is always another.

## Yearbook

Re-drawn by [the almost](persona_the_almost.md), who believes each new date as it is set.
It is the play's moving horizon: it turns [the sunk cost](piece_the_sunk_cost.md) from a loss into an investment-not-yet-returned, and as long as the finish is ahead the loss is only unrealised, so no gate has to be the one that realises it.
